# pfSense-pkg-SipRegistrar

Универсальный SIP-регистратор на базе Kamailio 6.1.1 для pfSense 2.7.2.

Превращает pfSense в лёгкий, не привязанный к производителю SIP-регистратор,
подходящий для небольших офисов, мастерских, складов и домашних установок.
Любые SIP-совместимые IP-телефоны, программные клиенты и шлюзы могут
регистрироваться на нём и совершать звонки по коротким номерам (2–5 цифр)
с опциональной маршрутизацией на внешние SIP-шлюзы по префиксу номера.

> Полная документация на английском: [README.md](README.md)

---

## Совместимость

### IP-телефоны и программные клиенты (любые SIP UA по RFC 3261)
- Grandstream GRP / GXP серии
- Yealink T2x, T4x, T5x серии
- Cisco CP-78xx, CP-88xx серии
- Fanvil X, H серии
- Snom 3xx, 7xx серии
- Программные клиенты: Zoiper, MicroSIP, Linphone, Bria и др.

### SIP-шлюзы (любые SIP-совместимые)
- Yeastar TA серия (FXO/FXS шлюзы)
- Grandstream GXW серия
- Cisco VG серия
- Audiocodes Mediant серия
- Asterisk / FreePBX / 3CX
- Любой другой SIP-шлюз или IP-АТС

## Схема сети (типовая установка)

```

  Аналоговые телефоны (любое количество)
        │
        └── SIP-шлюз (Yeastar TA800 / Grandstream GXW / Cisco VG)
                │   IP: задаётся в настройках пакета
                │
                └── Локальная сеть (любая подсеть)
                        │
                        ├── IP-телефоны / программные клиенты
                        │   Номера: 2–5 цифр (10..99999)
                        │   SIP ID: любой текст (alice, 101, director, ...)
                        │
                        ├── Дополнительные SIP-шлюзы (опционально)
                        │   Маршрутизация по префиксу номера
                        │
                        └── pfSense (Kamailio SIP-регистратор)
                            LAN IP = SIP realm (определяется автоматически)
                            SIP-порт: настраивается (по умолчанию 5060/UDP)

```

## Требования

| Компонент        | Версия                               |
|------------------|--------------------------------------|
| pfSense          | 2.7.2-RELEASE (amd64)                |
| FreeBSD          | 14.0-RELEASE (база)                  |
| Kamailio         | 6.1.1 (FreeBSD quarterly репо)       |
| PHP              | 8.2.x                                |

> **Важно**: Kamailio отсутствует в репозитории pfSense. Установщик скачивает
> его из стандартного FreeBSD quarterly репозитория через обходной механизм
> (`pfSense-Extra.conf`). При этом обновляются системные пакеты pfSense
> (php82, perl5, kea, rrdtool, libxml2 и другие). Подробности о рисках —
> в `SECURITY.ru.md`, инструкция по установке — в `INSTALL.ru.md`.

> **ASLR необходимо отключить** на хосте pfSense перед запуском Kamailio 6.x.
> Установщик (`install.sh`) делает это автоматически.

## Схема нумерации

- **Number (Номер)**: 2–5 цифр, без ведущего нуля, диапазон 10..99999.
  Примеры: `42`, `101`, `8500`, `10001`. Используется для набора.
- **SIP ID**: любой текст из латинских букв, цифр, подчёркивания,
  точки и дефиса (длина 1..64). Примеры: `alice`, `101`, `director`,
  `gw_ta800`. Используется как имя регистрации SIP на телефоне.
- Хеш пароля HA1 вычисляется как `md5(SIP ID + Realm + Password)` и
  хранится в `config.xml` pfSense. Открытый пароль не сохраняется.

Связь **Number → SIP ID** хранится в таблице `dbaliases` Kamailio.
Когда кто-то набирает Номер, Kamailio ищет алиас и пересылает INVITE
на текущую регистрацию соответствующего SIP ID.

## Быстрый старт

1. **Установите пакет** 
    Скопируйте пакет на pfSense. Например в каталог /tmp.
    Подключитесь по SSH к pfSense и выполните скрипт установки из каталога /tmp/offline командой sh install.sh.

2. **Откройте порт в брандмауэре**:
   *Firewall → Rules → LAN → Add* → UDP, destination port 5060
   (или выбранный вами), source `LAN net`.

3. **Настройте регистратор**:
   *Services → SIP Registrar*.
   - **Вкладка Settings**: проверьте SIP Port и SIP Realm (LAN IP
     подставляется автоматически). Выберите язык (English / Русский).
   - **Вкладка Gateways**: добавьте локальные FXS/FXO-шлюзы (например Yeastar TA810)
     с опциональными префиксами номеров. Аналоговые шлюзы, чьи FXS-линии
     регистрируются напрямую на Kamailio, ещё и определяются автоматически и
     показываются только для чтения (модель и живой статус — из их регистраций,
     без SNMP).
   - **Вкладка Trunks**: добавьте линии внешних SIP-провайдеров (например Ростелеком, Zadarma).
     Kamailio регистрируется как UAC-клиент. Бейджи показывают статус регистрации.
   - **Вкладка Devices**: добавьте телефоны и программные клиенты.
     Задайте Number, SIP ID и SIP Password.
   - **Вкладка Incoming**: настройте маршрутизацию DID от транков/шлюзов к добавочным
     с расписанием рабочего/нерабочего времени.
   - **Вкладка Outbound**: настройте какие добавочные могут совершать исходящие звонки
     через каждый транк или шлюз.
   - **Вкладка Groups**: ring/hunt-группы — параллельный обзвон или последовательный
     перебор с таймаутом на участника и fallback-номером.
   - **Вкладки Status / History**: живые регистрации и активные звонки; история
     вызовов (CDR) с фильтром по номеру и сортируемыми столбцами.
   - Нажмите **Сохранить** — изменения настроек/телефонов применяются без
     сброса регистраций. Изменения маршрутизации (Транки/Incoming/Outbound/
     Шлюзы) перезапускают Kamailio (route-блоки читаются только при старте), но
     регистрации сохраняются (db_text usrloc) и загружаются обратно — телефоны
     **не** слетают и не перерегистрируются.

4. **Настройте каждый телефон**:
   - SIP Server: LAN IP pfSense
   - SIP Port: как в Settings (по умолчанию 5060)
   - Username: SIP ID из таблицы Devices
   - Password: SIP Password из таблицы Devices

5. **Проверьте** в *Services → SIP Registrar → Status* —
   зарегистрированные телефоны появятся в течение 30–60 секунд.

Подробные инструкции — в `INSTALL.ru.md`.
